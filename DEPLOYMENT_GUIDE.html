SHL Dashboard Deployment Guide

Document Version: 1.0
Created: January 22, 2026
Purpose: Step-by-step instructions to deploy the Sonic Healthcare (SHL) Analysis Dashboard to GitHub and Vercel

Prerequisites

  • A GitHub account (free tier is sufficient)
  • A Vercel account (free tier is sufficient - can sign up with GitHub)
  • Web browser (Chrome, Safari, Firefox, or Edge)
  • The index.html dashboard file located at: 
    /Users/davidcargill/Library/CloudStorage/Dropbox/Side Mission Control/Stocks, Shares, ETF, Commodities/Research/Sonic Health Care/shl-dashboard/index.html

Part 1: Create GitHub Repository

Step 1.1: Sign in to GitHub

  1. Open your web browser and go to https://github.com
  2. Sign in with your GitHub credentials
  3. If you don't have an account, click "Sign up" and follow the registration process

Step 1.2: Create a New Repository

  1. Click the + icon in the top right corner of GitHub
  2. Select "New repository" from the dropdown
  3. Fill in the repository details:
    Field Value
    Repository name shl-dashboard
    Description Sonic Healthcare (SHL.AX) Financial Analysis Dashboard
    Visibility Public (required for free Vercel hosting)
    Initialize with README Leave unchecked
  4. Click "Create repository"

Step 1.3: Upload the Dashboard File

  1. On your new repository page, click "uploading an existing file" (in the quick setup section)
  2. Drag and drop the index.html file from: 
    Dropbox/Side Mission Control/Stocks, Shares, ETF, Commodities/Research/Sonic Health Care/shl-dashboard/
  3. Alternatively, click "choose your files" and navigate to select the file
  4. In the "Commit changes" section at the bottom:
    • Commit message: Initial SHL dashboard deployment
  5. Click "Commit changes"
Note: Your repository should now contain one file: index.html. The URL will be something like: https://github.com/YOUR_USERNAME/shl-dashboard

Part 2: Deploy to Vercel

Step 2.1: Sign up/Sign in to Vercel

  1. Go to https://vercel.com
  2. Click "Sign Up" or "Log In"
  3. Select "Continue with GitHub" (recommended - this links your accounts)
  4. Authorize Vercel to access your GitHub account when prompted

Step 2.2: Import Your GitHub Repository

  1. Once logged in, click "Add New..." button
  2. Select "Project"
  3. You'll see a list of your GitHub repositories
  4. Find shl-dashboard and click "Import"

Step 2.3: Configure Project Settings

  1. Project Name: shl-dashboard (or your preferred name)
  2. Framework Preset: Select "Other" (this is a static HTML site)
  3. Root Directory: Leave as ./ (default)
  4. Build Command: Leave empty (no build needed)
  5. Output Directory: Leave as default or set to ./
  6. Click "Deploy"

Step 2.4: Wait for Deployment

  1. Vercel will build and deploy your project (takes 30-60 seconds)
  2. You'll see a success message with confetti animation!
  3. Your dashboard is now live!

Part 3: Access Your Dashboard

Your Dashboard URL

After deployment, your dashboard will be available at:

https://shl-dashboard.vercel.app

Or a URL like:

https://shl-dashboard-YOUR_USERNAME.vercel.app

The exact URL is shown on your Vercel project dashboard.

Part 4: Custom Domain (Optional)

Step 4.1: Add a Custom Domain

  1. In your Vercel project dashboard, click "Settings"
  2. Navigate to "Domains"
  3. Enter your custom domain (e.g., shl.yourdomain.com)
  4. Follow the DNS configuration instructions provided

Part 5: Future Updates

How to Update Your Dashboard

  1. Go to your GitHub repository: https://github.com/YOUR_USERNAME/shl-dashboard
  2. Click on index.html
  3. Click the pencil icon (Edit this file) in the top right
  4. Make your changes in the editor
  5. Scroll down and click "Commit changes"
  6. Vercel will automatically redeploy within 30-60 seconds!

Important Notes

  • Auto-Deploy: Any changes pushed to GitHub will automatically trigger a new Vercel deployment
  • Free Tier Limits: Vercel's free tier includes 100GB bandwidth/month - more than enough for personal use
  • SSL/HTTPS: Vercel automatically provides free SSL certificates
  • No Server Required: This is a static site - no backend maintenance needed

Troubleshooting

Issue Solution
Repository not appearing in Vercel Click "Adjust GitHub App Permissions" and grant access to the repository
Deployment failed Check that your file is named exactly index.html (lowercase)
Page shows raw HTML Ensure the file has the correct .html extension
Charts not loading Check browser console for errors; ensure internet connection for CDN resources
Styles look broken Clear browser cache (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)
